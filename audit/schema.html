数据结构说明 · Schema

本页用自然语言解释三大产物中的每个字段,包括字段意义、可能的值、示例,以及为什么这个字段会存在。

1. 图书馆 Libraries

  • id:库的短代码,如 wakefield。所有交叉引用都用它。
  • name:正式馆名,如 "Lucius Beebe Memorial Library"。
  • town · network:所在市镇 / 隶属网络(NOBLE / MLN / Minuteman 等)。
  • platform:博物馆通行证后台,目前有三种 — Assabet(52 馆)/ LibCal(5 馆)/ MuseumKey(2 馆)。决定了我们用哪个 scraper。
  • card_page:办卡说明页 URL。
  • eligibility:办卡资格 — open_ma_resident 表示麻州居民均可办,residents_only 表示仅本镇居民。
  • supports_availability:平台是否可查未来 30 天的库存(MuseumKey 不支持)。
  • address:街道地址,5 个字段。
  • geo:经纬度,由 OSM Nominatim 反查得到,缓存在 data/.cache/geocode.json。

2. 景点 Attractions

  • slug:URL 友好的稳定 ID,如 boston-childrens-museum。
  • museum_name:正式名称。
  • address / website / phone / description:景点元数据,来自景点官网或 Wikipedia。
  • categories:分类标签数组(Children / Family / Science / Art / Nature / History ...)。
  • sources:提供本景点 pass 的所有图书馆 id 数组。
  • original_price:景点门市原价,两层模型。
    • age_pricing:按年龄定价,任何符合年龄的访客都适用。包含 adult / youth / child / senior 四种(每种是 {price, min_age, max_age} 或 null),以及 free_under_age(低于该岁数免费,是年龄阈值数字而非票价)。
    • identity_pricing:按身份定价,需出示证件。包含 student / educator / military 三种(每种是 {price, requires} 或 null)。
    • family:家庭通票价(顶层数字)。notes / source_url:备注与数据源。
  • hero_image:景点官网 <meta property="og:image"> 抓取的封面图;local_path 是本地缓存路径(gitignored)。
  • hours:周一至周日营业时间;status 字段表示数据可信度(ok / varies / seasonal / missing)。
  • geo:经纬度,用于距离排序。

3. 优惠规则 Passes

每行表示一个 (lib × attraction) 组合,共 1008 行。

  • library_id / attraction_slug:主键,联合唯一。
  • pass_type:三种 pass 形态 —
    • digital:邮件即时发送的电子券,用户可立即下载并使用(553 个)。
    • physical-coupon:门店取纸质券,用完不归还(260 个)。
    • physical-circ / loan-card:循环借阅卡,用完需归还图书馆(172 个)。
    • unknown:未能识别(23 个)。
  • discount:折扣类型与展示标签。class 五种:
    • free:完全免费(277 个)
    • half:半价 / 50% off(290 个)
    • percent-off:其它百分比折扣(4 个)
    • dollar-off:固定金额减免,如 $5 off(19 个)
    • price:打到一个固定低价,如 "Adults $5 / Kids $3"(342 个)
    • discount:笼统折扣(23 个);unknown:未识别(53 个)
  • policy:从 raw 文本中抽出的结构化字段:
    • max_people(总人数上限) / max_adults / max_children
    • free_under_age:低于该岁数免费
    • discount_percent / discount_dollar_off / savings_per_person_usd:折扣量
    • eligibility_tags:适用人群标签(见下)
    • exclusions:限制条件(见下)
    • boosts:额外福利
    • raw:原始优惠说明,所有抽取都基于它
  • source_phrases(在 raw/pass_policies/*.json 中):每个抽取字段对应在 raw 中的子串,供审计高亮。
  • availability:未来 30 天的可预订状态字典(museumkey 不可用)。

Eligibility 标签 · 13 种

  • all · open to all — 默认值,任何持卡人都能用
  • vehicle · per vehicle — 按车而非按人(适用州立公园)
  • single_ticket · 1 ticket — 单张票,而非"上限 N 人"
  • family · family pass — family pass,常含父母 + 子女
  • groups · group rate — 按团体打折
  • residents_only · residents only — 仅特定市镇居民
  • adults_only / children_only — 只覆盖成人或儿童一种票种
  • seniors_free · students_only · military_free · educator_free · members_free — 该群体特殊免费

Exclusions 限制

  • weekdays_only / weekends_only / weekends_excluded:工作日 / 周末限制
  • blackout_dates:特定日期不可用
  • reservation_required:必须先预约
  • id_required:入场需出示证件
  • seasonal:X-Y:仅在某月份范围内开放,例如 seasonal:apr-oct

Boosts 加成

  • library_card_required:进入景点需出示图书馆卡
  • members_discount:景点会员另享折扣
  • gift_shop_discount:礼品店折扣
  • ebt_discount / snap_free:EBT / SNAP 低收入援助
