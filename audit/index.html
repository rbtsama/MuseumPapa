数据质量信号 · 优先抽查项

这一节列出 AI 抽取过程中可能值得人工抽查的"可疑点"。每条都说明了"是什么/含义/怎么追溯"。 数字旁的链接跳转到相应专题页,可逐条核对 raw 原文与 AI 输出。

  • 0 条 pass 没有 coupon 字段 — 含义:plan-9 再抽取未覆盖到这些行,需补跑 subagent — see Policies
  • 3 条 pass 的 coupon.summary 为空(含 coupon 字段缺失的) — 含义:AI 抽出了结构但没生成摘要文字,前端 fallback 会显示 '(no extraction)'
  • 0 条 pass 的 pass_type 是 unknown(没归类成 digital/physical/loan-card 中的任一个) — 含义:从原始 HTML 抓不到明确的 pass 类型标识符
  • 0 对 attraction slug 重复(同一景点被存了两条记录) — 含义:数据建模历史遗留 bug,前端会显示成两张不同的卡,需合并 — see Duplicates
  • 22 个景点没有任何价格层级(adult/child/youth/senior/...)— 含义:景点官网无公开标价,或票价是 per-show / per-vehicle 浮动 — see Gaps
  • 4 个景点没有简介文字 — 含义:景点官网无 og:description / about 页 抓取失败 / 网站 403 拦截
