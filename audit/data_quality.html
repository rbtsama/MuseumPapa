Data quality数据质量 · 我们到底审了什么

What this page audits这页在审什么 · 一句话讲清

The 1026 passes in this dataset went through a 5-stage pipeline (see Lineage). At each stage, something can go wrong — a library page changes shape, the AI extractor misreads a sentence, a slug gets renamed without re-wiring. This page lists every category where that has actually happened, with a count and the affected rows.

简而言之,这页在审三件事: ① 每条记录是否完整(coupon 空不空、source_url 在不在)、 ② 字段值是否前后一致(标着 Adult 却写 12 岁以下、价格出现负数)、 ③ 我们抽出来的数字原文是否真的支持(原文写 "discounted admission" 不能被抽成具体百分比)。 数据 gaps(哪些字段还没抓到)放在本页最后一节。

HIGH Orphan raw coupon files (slug renamed but not re-wired) 0 孤立的原始 coupon 文件 · slug 改名后没接回

When an attraction's canonical slug changes, old extraction files can be left dangling. Each orphan = one real library discount that exists upstream but isn't visible on the live site. Should stay at 0.
景点 slug 改名后,旧的抽取文件没被接回主表,等于这条真实存在的优惠在产品里消失了。必须为 0。

MED Audience label contradicts its age range 0 受众标签和年龄区间矛盾 · 几乎都是抽取出错

An 'Adult' tier marked for under-13s, or a 'Child' tier for 18+, almost certainly means the extractor put the wrong row in the wrong bucket. UI ends up showing nonsensical age bands.
标着 Adult 却写 13 岁以下,或者 Child 却写 18 岁以上,几乎肯定是抽取放错位置。前端会显示前后矛盾的年龄区间,直接打击用户信任。

AI 抽取器在这些文件上放弃了,对应的优惠从来没进过产品。每多一条就少一条用户能看到的折扣。要么重跑,要么人工补录。

LOW Discount marked but no number — upstream is vague 90 标着有折扣但抽不到数值 · 上游原文本来就模糊

The library wrote things like 'discounted admission' or 'family rate' with no number. The discount is real but we can't show '$X off' or '50% off' to the user. Not a bug — an upstream limitation.
图书馆原文写的是 'discounted admission' / 'family rate' 这种模糊语,折扣是真的存在,但拿不到具体数字。属上游限制不是抽取 bug,前端只能显示"有折扣"。
