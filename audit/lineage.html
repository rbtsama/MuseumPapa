1

Collect from library websites从图书馆网站采集

我们按计划访问每家图书馆的 museum-pass 页面,把原始 HTML 和列表数据原样存档,作为后续抽取的唯一真相来源。

In
Library pass-program pages — public URLs we visit on a schedule
We do
Scrape every page, save the raw HTML and the listing JSON
Out
Raw page snapshots, one folder per library
59 library websites scraped

2

Catalog the attractions建立景点目录

把不同图书馆对同一家博物馆的不同写法合并成一条规范记录,再从景点官网补齐电话、地址、营业时间、票价、封面图,形成统一目录。

In
Raw library listings naming hundreds of museums, parks, theaters
We do
Deduplicate name variants into one canonical attraction record per real-world venue, enrich with phone / address / hours / hero image / price from each attraction's official site
Out
One row per unique attraction
97 unique attractions catalogued
Technical lineage技术血缘 · 给审计 / 工程团队
Entry
python scripts/build.py (catalog + attractions phases)
Modules
src/malibbene/build/catalog.pyMerge 3 platform raw indexes into one nested library_catalog.json
src/malibbene/build/slug_canonical.pyLegacy-slug → canonical-slug mapping (e.g. museum-of-fine-arts → mfa)
src/malibbene/build/attractions.pyGroup catalog passes by canonical slug, attach price/image/geo/hours enrichments
src/malibbene/build/categories.pyNormalize 21 raw category labels into a 9-bucket scheme
src/malibbene/common/normalize.pyLex-table that turns free-text benefit strings into short labels (FREE / 50% off / $5 off / discount / unknown)
How combined
library_catalog.json is keyed by lib_id; attractions.py inverts it — for each canonical slug it collects every library that lists that slug, then merges in side-files data/raw/attractions/{prices,images,geo,hours}/<slug>.json produced by the attraction-side scrapers and subagent extractors.
Output path
data/structured/library_catalog.json · data/structured/attractions.json
Audit hooks
Slug-collision check: see n_unmapped_passes_per_platform in catalog meta. If any platform reports unmapped passes, the slug map missed a legacy alias. Attractions _meta.n_with_price uses _has_numeric_price() — a non-empty original_price object alone does NOT count.
3

Structure each pass into a clean record把每张 pass 整理成结构化记录

每家图书馆描述福利的文字风格都不一样,我们把它们翻译成统一的字段:覆盖人群、折扣形式、取卡方式、使用限制,让前端可以横向对比。

In
Free-text benefit descriptions written by each library in their own style
We do
Read the natural language and extract who the pass covers, what discount it gives, how it is delivered, and any usage restrictions
Out
One row per (library × attraction) combination, with a uniform discount summary
1026 attraction/library pass combinations
Technical lineage技术血缘 · 给审计 / 工程团队
Entry
python scripts/build.py (passes phase)
Modules
src/malibbene/build/passes.pyFlatten library_catalog into one row per (lib_id, canonical_slug); resolve pickup method and pass_type
src/malibbene/build/coupons.pyAttach capacity + audience_policies blocks from per-pass coupon files
src/malibbene/build/museum_policy.pyDrop audience_policies that merely restate the museum's own free-under-N policy (avoids double-counting)
data/raw/pass_coupons/{lib}_{slug}.json1008 per-pass coupon extractions produced by plan-9 Sonnet subagents reading the raw HTML — carries source_phrases for provenance
How combined
passes.py iterates every (lib_id × raw_slug) in the catalog, maps raw_slug → canonical via slug_canonical.py, looks up the coupon file with the raw key first (then canonical fallback), and emits one passes.json row. Vehicle capacity defaults n=1 in coupons.py when the extractor returned null.
Output path
data/structured/passes.json
Audit hooks
coupon.summary is intentionally NOT stored — it's derived on display via _derive_coupon_summary() (build_audit_site.py line ~723). Re-extraction status of edge cases lives in data/structured/_audit_unspecified_reclassification.json (16 corrected, 35 by-audience, 9 family, 45 vague).
4

Attach live booking availability接入实时预订日历

每天自动刷新近期预订位,用户打开页面看到的是接近实时的可借状态,不是静态截图。

In
Each library's reservation system (Assabet, LibCal, MuseumKey)
We do
Refresh near-term slots daily so users see what is bookable right now
Out
Per-pass availability calendar refreshed on a schedule
975 with live booking calendar
Technical lineage技术血缘 · 给审计 / 工程团队
Entry
python scripts/scrape_dynamic.py
Modules
src/malibbene/sources/assabet/availability.pyWalks 3 month-pages per pass (base/ · next/ · next/next/), parses day-has-openings / day-no-openings / time-partially-available CSS classes
src/malibbene/sources/libcal/availability.pyCalls LibCal's institution endpoint with the current and next month's first-day-of-month parameter; collapses 3 LibCal states to available / booked
(MuseumKey not scraped)Cohasset + Hingham require library-card login; documented as catalog-only in BRD §A.3
How combined
Per-pass calendar dicts (YYYY-MM-DD → available|limited|booked) are written to data/raw/{assabet,libcal}/availability/{lib_id}.json, then merged into passes.json's availability field at next build. ~2900 HTTP requests per full refresh; 24h cache TTL short-circuits within-day re-runs.
Output path
data/raw/{assabet,libcal}/availability/{lib_id}.json → passes.json:availability
Audit hooks
Front-end's AttractionDetail.tsx uses an explicit-only predicate (availability[date] === 'available') — undefined does NOT default to available, so unscraped months stay blank instead of faking green. Month-pill row is capped at dataHorizon (max date across all this attraction's passes).
