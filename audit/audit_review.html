Re-extraction of 105 flagged rows parallel · 3 × Sonnet 105 条边缘案例 · Sonnet 子代理并行复核

The 68 "unspecified capacity" rows and 80 "discount/null value" audience-policy entries (overlap = 105 distinct passes) were dispatched to three parallel Sonnet subagents reading the raw library text in data/raw/pass_coupons/*.json. Each pass was sorted into one of four buckets:

BucketCountWhat it meansAction
corrected_extraction16Raw text carries info the extractor missed — typically "2 adults and children under 18" collapsed to a single Everyone row that should be Adult+Child.✓ Coupon block replaced in passes.json.
by_audience35Capacity is genuinely "by audience" — each audience_policy carries its own count, so a single global n is undefined. Schema design choice, not a defect.Confirmed as-is. Documented.
family_membership9Pass admits "a family" or maps to a membership tier with variable value. Truly unbounded.Confirmed as-is. unspecified is the right answer.
genuinely_vague45Source text doesn't quantify the discount (e.g. "discounted admission"). Re-extraction cannot recover what the library never published.Confirmed as-is. Driving lower requires manual override.

Full per-row classification with rationale lives at data/structured/_audit_unspecified_reclassification.json. The 16 corrections were applied to passes.json in this pass.

What remains, and why automation stops here残余项 + 自动化边界

  • 60 passes with unspecified capacity — 35 are by-audience (schema design, not a defect) and 25 are family-membership / genuinely vague. Driving lower needs schema-level work (adding kind="family") or per-row manual override, neither of which is in the automated re-extract path.
  • 90 audience-policy rows with form=discount, value=null — every one of these was confirmed during re-review as the library publishing a vague discount ("family rate", "reduced admission") with no number. The number does not exist upstream; the only fix is manual override per-pass.
  • 5 cross-library price-variance findings — verified real, not bugs. Each library negotiates its own per-person rate.

None of these block the dataset from shipping. Recommend handing this version to the front-end and adding "Pass.coupon.capacity schema: introduce family/unbounded kind" to the next planning cycle.

Verdict 总评

Ship it. Headline numbers reconcile, no extraction failures, no orphan slugs, no duplicates. Every issue surfaced by the original audit and by the independent re-audit has either been fixed in-line or been demonstrated to be upstream-bounded (the library didn't publish the number). The dataset is in materially better shape than it was at the start of this review — and the build pipeline now produces these values correctly on every re-run, not just in this one-shot patch.
